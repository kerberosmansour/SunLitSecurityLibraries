## Key Design Decisions

### 1. cargo-deny 0.19 Schema Changes

The runbook's deny.toml template was written for cargo-deny 0.18. Version 0.19 introduced breaking changes:
- `[advisories]` dropped `vulnerability`, `unmaintained`, `unsound`, `yanked`, `notice` fields; these are now defaults handled internally
- `[licenses]` dropped `unlicensed` and `copyleft` fields; enforcement is now purely via the `allow` list
- `[bans]` `wildcards` field now accepts "allow"/"warn"/"deny" (not "all"/"workspace"/"transitive"/"none")
- `[[licenses.exceptions]]` syntax changed to inline `exceptions = [{ allow = [...], crate = "..." }]`

**Correction for future milestones**: When the runbook template and tool version conflict, the tool version wins. Always generate a reference template with `cargo deny init` or check `deny.template.toml` in the cargo-deny source.

### 2. Advisory Ignore List

`lru v0.12.5` has RUSTSEC-2026-0002 (IterMut stacked-borrows). Since M10 has no refactor budget, the advisory was added to the ignore list with justification. Future work should upgrade `lru` or replace it with a maintained alternative.

### 3. License Exceptions for Transitive Deps

- `r-efi` (LGPL-2.1-or-later): Transitive dependency of `getrandom` for UEFI platform targets. Not linked into our Linux/macOS/Windows output.
- `smartstring` (MPL-2.0): Used by `rhai` for string internment. MPL-2.0 is file-level copyleft; no MPL-2.0 files were incorporated into our source.

### 4. Workspace Crate License Metadata

Our 9 workspace crates lacked `license` fields in their Cargo.toml. cargo-deny 0.19 without `[licenses.private] ignore = true` reports them as "unlicensed". Adding `license = "MIT"` to each Cargo.toml resolved this. This is a metadata addition, not a source code change.

### 5. cargo-vet Bootstrap Strategy

248 third-party dependencies were all exempted during `cargo vet init`. This is the standard bootstrap approach — all existing dependencies get exemptions at the `safe-to-deploy` or `safe-to-run` criteria level. New dependencies added after this point will require auditing or explicit exemption.
