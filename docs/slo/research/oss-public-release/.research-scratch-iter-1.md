## Initial Findings

- **Trusted publishing (Q1):** A `crates.io` "Trusted Publishing" feature using GitHub OIDC was announced in mid/late 2025; status as of April 2026 is plausibly GA but **needs verification** before being mandated. Confidence: medium.
- **`actions/attest-build-provenance` (Q2):** Produces Sigstore-signed in-toto SLSA provenance v1 attestations stored in Rekor and queryable via `gh attestation verify`. Whether this *alone* satisfies SLSA Build L3 (vs. requiring the SLSA reusable-workflow generator like `slsa-framework/slsa-github-generator`) is contested in the community: GitHub blog posts position it as L3-capable, while strict readings of the SLSA v1.0 spec require an isolated, non-falsifiable build that the reusable workflow more clearly provides. **Needs verification** with current SLSA spec text and GitHub's threat-model doc. Confidence: medium-low.
- **OSS-Fuzz Rust (Q3):** OSS-Fuzz has supported Rust via `cargo-fuzz` for several years; onboarding requires a `projects/<name>/{project.yaml, Dockerfile, build.sh}` PR plus a primary contact with a verifiable affiliation. Turnaround historically 1–3 weeks. Existing in-tree `libfuzzer-sys` harnesses (already present in this repo, see Repo Context) are directly reusable. Confidence: high on shape, medium on 2026-current turnaround.
- **CodeQL Rust (Q4):** CodeQL added Rust support in public preview around 2024–2025. Whether it has reached GA by April 2026 is **needs verification**. Even in preview, CodeQL Rust uploads to the Security tab and supports GitHub Advanced Security; rule coverage is thinner than Java/Go. Confidence: medium.
- **License default (Q5):** `MIT OR Apache-2.0` remains the dominant Rust convention (it is what `cargo new` recommends and what `rust-lang/*` crates use); no significant movement away from it as of late 2025. The Apache-2.0 patent grant is the main reason for dual-licensing over MIT-only. Confidence: high (subject to verification for any 2026 ecosystem shift).
- **cargo-vet imports (Q6):** Mozilla (`https://hg.mozilla.org/mozilla-central/raw-file/tip/supply-chain/audits.toml`), Google's Fuchsia/Chromium feeds, Bytecode Alliance, ISRG, and Embark have all historically published vet-compatible audit feeds. Canonical URLs and current cargo-vet 0.10 compatibility **need verification** — URLs have moved before. Confidence: medium.
- **cosign verify UX (Q7):** Keyless verification still requires `--certificate-identity` (or `--certificate-identity-regexp`) and `--certificate-oidc-issuer https://token.actions.githubusercontent.com` for GitHub-built artifacts. `cosign verify-blob --bundle` simplifies the bundle-attached flow. Alternative: `gh attestation verify` consumes the same Sigstore bundle without a cosign install. Confidence: high.
- **harden-runner (Q8):** `step-security/harden-runner` is Linux-only on GitHub-hosted runners; macOS/Windows are not supported. Audit-mode → block-mode transition typically takes 2–3 baseline runs across PR + main pushes. Confidence: medium (Linux-only restriction is known; baseline duration needs verification).
- **dependency-review-action (Q9):** Cargo lockfile parsing was added in 2023 and is reported GA; license/advisory policy uses a YAML config-file format. Confidence: medium.
- **Scorecard ceiling (Q10):** Realistic ceiling for a solo-maintainer Rust repo is ~8/10; checks that commonly score low for solo projects: `Code-Review` (PRs are often self-merged), `Branch-Protection` (until configured), `Token-Permissions`. Top score ≥9 generally requires multi-maintainer review. Confidence: high on the rough ceiling, medium on exact thresholds.
- **MSRV (Q11):** Mainstream "modern" libraries in 2025–2026 sit around 1.74–1.80 (`async-fn-in-trait` stable in 1.75; `LazyLock`/`LazyCell` stable in 1.80). A defensible MSRV for a security library wanting a 6–12 month tail is roughly N-3 to N-6 stable releases. Specific number **needs verification** against current stable. Confidence: medium.
- **GHSA + RUSTSEC (Q12):** Canonical practice as of late 2025 is "publish a GHSA via private vuln reporting; RUSTSEC advisory is opened by RustSec WG (often automatically from the GHSA)." Needs verification that auto-import is now reliable. Confidence: medium.
